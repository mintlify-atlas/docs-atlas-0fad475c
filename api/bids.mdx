## Overview

The bid endpoints provide read access to bidding activity, including recent bids across all teams and rebid marketplace offers during Round 4. These endpoints support the live bid feed and marketplace UI.

## Get recent bids

### GET /api/data/bids/recent

Retrieve the most recent bids across all plots and teams.

```bash
curl http://localhost:8000/api/data/bids/recent
```

**Query parameters:**

<ParamField query="t" type="number">
  Cache-busting timestamp to prevent stale data
  
  Example: `t=1709123456789`
</ParamField>

<ParamField query="limit" type="number" default="10">
  Maximum number of recent bids to return
  
  Example: `limit=20`
</ParamField>

**Response:**

<ResponseField name="bids" type="array">
  Array of bid objects, sorted by timestamp (newest first)
  
  <Expandable title="Bid object structure">
    <ResponseField name="id" type="string">
      Unique bid identifier
    </ResponseField>
    
    <ResponseField name="plot_number" type="number">
      Plot number this bid is for
    </ResponseField>
    
    <ResponseField name="team_id" type="string">
      ID of the team that placed the bid
    </ResponseField>
    
    <ResponseField name="team_name" type="string">
      Display name of the bidding team
    </ResponseField>
    
    <ResponseField name="amount" type="number">
      Bid amount in rupees (₹)
    </ResponseField>
    
    <ResponseField name="timestamp" type="string">
      ISO 8601 timestamp of when the bid was placed
    </ResponseField>
    
    <ResponseField name="round" type="number">
      Auction round when bid was placed (1, 4, etc.)
    </ResponseField>
  </Expandable>
</ResponseField>

**Example response:**

```json
[
  {
    "id": "bid-12345",
    "plot_number": 5,
    "team_id": "team-2",
    "team_name": "Beta Builders",
    "amount": 1500000,
    "timestamp": "2026-03-02T14:23:45.123Z",
    "round": 1
  },
  {
    "id": "bid-12344",
    "plot_number": 5,
    "team_id": "team-1",
    "team_name": "Alpha Developers",
    "amount": 1450000,
    "timestamp": "2026-03-02T14:23:12.456Z",
    "round": 1
  }
]
```

**Usage example:**

```tsx app/dashboard/page.tsx
const bidsRes = await fetch(`/api/data/bids/recent?t=${Date.now()}`, {
  cache: "no-store"
});
const recentBids = await bidsRes.json();
```

## Get rebid marketplace offers

### GET /api/data/rebid-offers

Retrieve active and sold rebid offers from the Round 4 marketplace.

```bash
curl http://localhost:8000/api/data/rebid-offers
```

**Query parameters:**

<ParamField query="t" type="number">
  Cache-busting timestamp
</ParamField>

<ParamField query="status" type="string">
  Filter by offer status:
  - `"active"`: Currently listed offers
  - `"sold"`: Completed transactions
  - Omit to get all offers
</ParamField>

**Response:**

<ResponseField name="offers" type="array">
  Array of rebid offer objects
  
  <Expandable title="Offer object structure">
    <ResponseField name="id" type="string">
      Unique offer identifier
    </ResponseField>
    
    <ResponseField name="plot_number" type="number">
      Plot being offered for resale
    </ResponseField>
    
    <ResponseField name="seller_team_id" type="string">
      ID of team selling the plot
    </ResponseField>
    
    <ResponseField name="seller_team_name" type="string">
      Display name of seller team
    </ResponseField>
    
    <ResponseField name="asking_price" type="number">
      Listed price in rupees (₹)
    </ResponseField>
    
    <ResponseField name="original_price" type="number">
      Original purchase price before adjustments (₹)
    </ResponseField>
    
    <ResponseField name="markup_percent" type="number">
      Markup percentage: `(asking_price - original_price) / original_price * 100`
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Offer status:
      - `"active"`: Available for purchase
      - `"sold"`: Successfully sold
      - `"cancelled"`: Removed from marketplace
    </ResponseField>
    
    <ResponseField name="buyer_team_id" type="string">
      ID of purchasing team (only if status is `"sold"`)
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp when offer was created
    </ResponseField>
    
    <ResponseField name="sold_at" type="string">
      ISO 8601 timestamp when offer was sold (only if sold)
    </ResponseField>
  </Expandable>
</ResponseField>

**Example response:**

```json
[
  {
    "id": "offer-123",
    "plot_number": 7,
    "seller_team_id": "team-1",
    "seller_team_name": "Alpha Developers",
    "asking_price": 1650000,
    "original_price": 1500000,
    "markup_percent": 10,
    "status": "active",
    "created_at": "2026-03-02T15:10:00.000Z"
  },
  {
    "id": "offer-122",
    "plot_number": 3,
    "seller_team_id": "team-3",
    "seller_team_name": "Gamma Group",
    "asking_price": 2200000,
    "original_price": 2000000,
    "markup_percent": 10,
    "status": "sold",
    "buyer_team_id": "team-2",
    "created_at": "2026-03-02T15:05:00.000Z",
    "sold_at": "2026-03-02T15:15:30.000Z"
  }
]
```

**Usage example:**

```tsx app/dashboard/page.tsx
const offersRes = await fetch(`/api/data/rebid-offers?t=${Date.now()}`, {
  cache: "no-store"
});
const rebidOffers = await offersRes.json();
```

## Get sold rebid offers

### GET /api/data/rebid-offers-sold

Retrieve only completed rebid transactions.

```bash
curl http://localhost:8000/api/data/rebid-offers-sold
```

**Query parameters:**

<ParamField query="t" type="number">
  Cache-busting timestamp
</ParamField>

**Response:**

Same structure as `/api/data/rebid-offers`, but filtered to only include offers with `status: "sold"`.

**Usage example:**

```tsx app/dashboard/page.tsx
const soldOffersRes = await fetch(
  `/api/data/rebid-offers-sold?t=${Date.now()}`,
  { cache: "no-store" }
);
const rebidOffersSold = await soldOffersRes.json();
```

## Rebid offer pricing rules

During Round 4 sell phase, teams can list plots with these constraints:

**Minimum price:**
```javascript
const currentValue = plot.current_bid + plot.round_adjustment;
const minPrice = currentValue;
```

**Maximum price (10% markup):**
```javascript
const maxPrice = currentValue * 1.1;
```

**Validation:**
```javascript
if (askingPrice < minPrice || askingPrice > maxPrice) {
  throw new Error(
    `Price must be between ₹${minPrice.toLocaleString()} and ₹${maxPrice.toLocaleString()}`
  );
}
```

## Real-time updates

Bid and offer data updates in real-time via Socket.IO:

**new_bid event:**
```tsx app/dashboard/page.tsx
socket.on("new_bid", (data) => {
  // Add to recent bids list
  setRecentBids(prev => [data, ...prev].slice(0, 10));
  
  // Update current plot's bid
  setCurrentPlot(prev => prev ? {
    ...prev,
    current_bid: data.amount,
    winner_team_id: data.team_id
  } : null);
});
```

**new_rebid_offer event:**
```tsx
socket.on("new_rebid_offer", (offer) => {
  setRebidOffers(prev => [offer, ...prev]);
});
```

**rebid_offer_sold event:**
```tsx
socket.on("rebid_offer_sold", (offer) => {
  // Remove from active offers
  setRebidOffers(prev => prev.filter(o => o.id !== offer.id));
  
  // Add to sold offers
  setRebidOffersSold(prev => [offer, ...prev]);
});
```

**rebid_offer_cancelled event:**
```tsx
socket.on("rebid_offer_cancelled", (offer) => {
  setRebidOffers(prev => prev.filter(o => o.id !== offer.id));
});
```

See [Socket.IO events](/api/socket-events) for complete documentation.

## Bid feed display

The bid feed component shows recent bidding activity:

```tsx components/BidCard.tsx
{recentBids.map(bid => (
  <BidCard
    key={bid.id}
    plotNumber={bid.plot_number}
    teamName={bid.team_name}
    amount={bid.amount}
    timestamp={bid.timestamp}
  />
))}
```

Bids are automatically sorted by timestamp (newest first) and limited to the 10 most recent bids.

## Error responses

<ResponseField name="error" type="object">
  <Expandable title="Error structure">
    <ResponseField name="detail" type="string">
      Human-readable error message
    </ResponseField>
    
    <ResponseField name="status_code" type="number">
      HTTP status code
    </ResponseField>
  </Expandable>
</ResponseField>

**Common errors:**

| Status | Error | Description |
|--------|-------|-------------|
| 400 | Invalid parameters | Invalid query parameters |
| 500 | Internal server error | Database connection or server issue |

**Example error:**

```json
{
  "detail": "Failed to retrieve bid data",
  "status_code": 500
}
```
