The public leaderboard page provides a dedicated, full-screen view of team rankings optimized for projection displays, streaming overlays, and audience screens during live auction events.

## Overview

Accessible at `/leaderboard`, this page displays real-time team rankings with automatic updates via Socket.IO. Unlike the live view page which shows the map and current plot, the leaderboard focuses entirely on team scores and standings.

## Scoring system

Team rankings are calculated using the final score formula:

```typescript Score calculation
const PLOT_BONUS = 10000000; // ₹10 Lakh per plot

// Calculate for each team:
const remainingCash = budget - spent;
const propertyValue = sum of (plot prices + adjustments) for owned plots;
const netWorth = remainingCash + propertyValue;
const finalScore = netWorth + (plotsWon * PLOT_BONUS);
```

<Note>
  The ₹10 Lakh bonus per plot incentivizes strategic plot acquisition beyond just maximizing cash reserves.
</Note>

## Real-time updates

The leaderboard connects via Socket.IO and listens for:

```typescript Socket events
// Team financial updates
socket.on('team_update', (data) => {
  // Update team's spent, budget, plots_won
});

// Plot ownership changes
socket.on('plot_update', (data) => {
  // Recalculate property values
});

// Auction state changes
socket.on('auction_state_update', (data) => {
  // Update auction status display
});
```

## Design considerations

### Responsive behavior
- **Desktop** - Full table view with all columns visible
- **Mobile** - Stacked cards with essential info only
- **Large displays** - Increased font sizes and spacing

### Performance
- **Throttled updates** - Rankings recalculate only when necessary
- **Memoized calculations** - Expensive score calculations are cached
- **Optimized rendering** - Only changed teams trigger re-renders
