## Join the auction

This guide walks you through logging in as a team, navigating the dashboard, and placing your first bid.

<Steps>

<Step title="Log in with your team credentials">

Navigate to the AuctionFest home page. You'll see a bold neo-brutalist login screen with:
- A large "PLANOMICS AUCTION" title
- Team name input field
- Passcode input field
- Connection status indicator

**Enter your credentials:**

1. Type your **team name** (case-insensitive matching)
2. Enter any **passcode** with 3+ characters for demo purposes
3. Click **ENTER AUCTION**

<CodeGroup>
```typescript Team authentication logic
// from app/page.tsx:40
const handleLogin = () => {
  if (!teamName.trim()) {
    setError("Please enter your team name");
    return;
  }
  // Match team by name (case-insensitive)
  const team = teams.find(
    (t) => t.name.toLowerCase() === teamName.trim().toLowerCase(),
  );
  if (!team) {
    setError("Team not found! Please check the team name.");
    return;
  }
  if (passcode.length < 3) {
    setError("Enter a valid passcode (any 3+ chars for demo)");
    return;
  }

  localStorage.setItem("team_id", team.id);
  router.push("/dashboard");
};
```
</CodeGroup>

<Warning>
  If you see "Team not found", verify your team name spelling with the admin. Team IDs are stored in `localStorage` for session persistence.
</Warning>

</Step>

<Step title="Explore your team dashboard">

Once logged in, you'll land on your team dashboard with three key sections:

### Header: Budget overview

Your financial status is always visible at the top:

- **Cash**: Remaining budget (Budget - Spent)
- **Plots**: Total value of owned plots (bid amount + adjustments)
- **Net Worth**: Cash + Plots


### Left panel: Plot visualization

Watch the current plot's SVG shape animate in with a countdown timer during "Going..." phase.

### Center panel: Plot details & bid controls

See base price, current bid, total area, and winner information.

### Right panel: Live bid feed

All bids for the current plot appear here in real-time.

<Info>
  The **Server Online** indicator at the top shows your Socket.IO connection status. Green = connected, Red = offline.
</Info>

</Step>

<Step title="Place your first bid">

### Understanding the bid interface

The bid card shows:

- **Current Highest Bid** or **Starting Price** (with adjustments applied)
- **Minimum bid**: Current bid + ₹1,00,000 (1 Lakh) increment
- **Your projected balance** if you win

### Bidding workflow

<Tabs>
  <Tab title="Step-by-step">
    1. Click the **+** button to increment your bid by ₹5,00,000 (5 Lakhs)
    2. Click **-** to decrement if needed
    3. The bid amount displays in Indian number format with Crore/Lakh labels
    4. Click **PLACE BID** when ready
  </Tab>
  <Tab title="Code reference">
    ```typescript
    // from components/BidCard.tsx:99
    const handlePlaceBid = () => {
      if (!socket || !currentPlot || !userTeam) return;

      // Prevent bids during adjustment rounds
      if (
        currentRound === 2 ||
        currentRound === 3 ||
        currentRound === 5 ||
        currentRound === 6
      ) {
        setError("Bidding not allowed during adjustment rounds");
        return;
      }

      // Prevent consecutive bids by same team
      if (currentPlot.winner_team_id === userTeam.id) {
        setError("You already hold the highest bid!");
        return;
      }

      // Require user to press + or - first to activate bid
      if (!isBidActive) {
        setError("Press + to place a bid");
        return;
      }

      const amount = isBidActive && bidAmount ? parseInt(bidAmount) : minBid;
      if (isNaN(amount) || amount < minBid) {
        setError(`Minimum bid amount is ₹${minBid.toLocaleString("en-IN")}`);
        return;
      }

      const remaining = Number(userTeam.budget) - Number(userTeam.spent || 0);
      if (amount > remaining) {
        setError(
          `Bid exceeds remaining budget (₹${remaining.toLocaleString("en-IN")})!`,
        );
        return;
      }

      setIsSubmitting(true);
      setError("");

      socket.emit("place_bid", {
        team_id: userTeam.id,
        amount: amount,
        plot_number: currentPlot.number,
      });
    };
    ```
  </Tab>
</Tabs>

### Real-time bid updates

When any team places a bid:

```typescript
// from app/dashboard/page.tsx:336
const handleNewBid = (data: any) => {
  setRecentBids((prev) => [data, ...prev].slice(0, 10));

  setCurrentPlot((prev: any) =>
    prev
      ? {
        ...prev,
        current_bid: data.amount,
        winner_team_id: data.team_id,
      }
      : null,
  );

  setPlots((prev) =>
    prev.map((p) =>
      p.number === data.plot_number
        ? { ...p, current_bid: data.amount, winner_team_id: data.team_id }
        : p,
    ),
  );
};
```

<Tip>
  **Pro tip:** You cannot bid consecutively. If you hold the highest bid, wait for another team to outbid you before placing another bid.
</Tip>

</Step>

<Tabs>

## Round 4 special: Rebid marketplace

In Round 4, teams can sell their owned plots:

<Steps>
  <Step title="View your portfolio">
    When Round 4's "Sell Phase" activates, you'll see all your owned plots.
  </Step>
  <Step title="List a plot for sale">
    - Enter asking price between **1x–1.1x** current value
    - Click **SELL** to list on marketplace
    - Other teams see your offer and can purchase instantly
  </Step>
  <Step title="Cancel listing">
    Click **UNSELL** to remove from marketplace before someone buys.
  </Step>
</Steps>

```typescript
// from app/dashboard/page.tsx:927
const currentValue =
  parseFloat(
    plot.current_bid || plot.total_plot_price || 0,
  ) + parseFloat(plot.round_adjustment || 0);
const maxAllowed = currentValue * 1.1; // 10% markup
const minAllowed = currentValue; // No discount
```

<Warning>
  **You cannot bid on plots you are selling.** The system will disable bidding if you're the original seller in Round 4.
</Warning>
